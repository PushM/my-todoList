# 多端同步说明

## 架构
- 电脑端: Electron 本地应用
- 手机端: `/mobile/` 下的可安装 PWA，Android 浏览器可“添加到主屏幕”
- 云端: `server/server.js` 提供同步 API 和手机端静态页面

## 同步方式
- 暂时不做账号登录
- 电脑端和手机端填写同一个“同步码”即可共享数据
- 同步策略为“整份状态文档按更新时间覆盖”
- 两端都有自动同步，间隔 30 秒，也支持手动同步

## 本地启动云端服务
```powershell
npm run start:server
```
默认监听 `8787` 端口。

## 部署到你的云服务器
1. 把整个项目上传到服务器
2. 服务器安装 Node.js 22+
3. 在项目目录执行 `npm install`
4. 执行 `npm run start:server`
5. 放通服务器安全组或防火墙端口 `8787`
6. 浏览器访问 `http://你的公网IP:8787/mobile/`

## 桌面端配置
- 服务器地址填 `http://你的公网IP:8787`
- 同步码自定义一个字符串，例如 `mytodo-001`
- 点击“保存同步设置”

## Android 端配置
1. 打开 `http://你的公网IP:8787/mobile/`
2. 在浏览器中选择“添加到主屏幕”
3. 打开安装后的应用
4. 填写同样的服务器地址和同步码
5. 点击“保存并同步”

## 注意
- 目前没有账号体系，同步码知道的人都能访问对应数据
- 当前冲突处理是“最后更新时间最新的一端覆盖旧数据”
- 如需更安全，可以下一步加 HTTPS、Nginx 反代、设备白名单或正式登录
