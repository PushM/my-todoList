# Android App

这个目录提供了一个原生 Android 壳，用来复用现有手机网页端界面，并把“已完成任务”同步到手机系统日历。

## 功能

- 使用 `WebView` 加载打包进 App 的手机端页面
- 继续使用现有服务端 `/api/sync` 同步待办数据
- 在 Android 端把已完成任务写入系统日历
- 监听网页端状态变化，当前端本机完成任务时立即写入日历
- 使用 `WorkManager` 周期拉取服务端状态，把其他设备上完成的任务同步到手机日历

## 重要限制

- Android 端必须显式填写 `服务器地址`，不能留空
- 周期后台同步依赖 `WorkManager`，最短周期是 15 分钟，不是秒级实时
- 首次启动需要授予日历权限，并选择一个写入目标日历

## 打开方式

1. 用 Android Studio 打开 [android](/D:/project/my-todoList/android)
2. 等待 Gradle 同步完成
3. 连接真机或启动模拟器
4. 运行 `app` 模块

## 使用步骤

1. 启动服务端：`npm run start:server`
2. 安卓端进入“同步设置”
3. 填写服务器地址，例如：`http://192.168.1.10:8787`
4. 填写和桌面端一致的同步码
5. 保存后授予日历权限
6. 选择一个系统日历作为写入目标

## 同步逻辑

- 安卓端本机完成任务：网页状态会立即传给原生桥接层，原生层直接更新系统日历
- 其他设备完成任务：Android 后台 worker 周期拉取服务端状态，再同步到系统日历
- 任务撤销完成或删除：Android 端会删除对应日历事件
- 完成日期调整：Android 端会更新对应日历事件日期
